- before committing, ensure you `mix format`, run the tests, run `mix test.integration`, run `mix docs`, run `mix credo` (elixir's linter) and refactor accordingly; always run integration tests as well before committing, and ensure documentation stays consistent
- for integration tests, prefer `mix test.integration` directly; start the relay yourself with `docker compose -f docker-compose.integration.yml up -d relay` after generating local certs via `scripts/generate_integration_certs.sh`, so the relay can stay up across repeated local test runs. If you regenerate the integration certs or hit TLS handshake errors like `invalid peer certificate: BadSignature`, recreate the relay container (`docker compose -f docker-compose.integration.yml down --remove-orphans && docker compose -f docker-compose.integration.yml up -d relay`) so it remounts the fresh certs instead of keeping stale container state
